## Milestone v1.1 Requirements

### CONTEXT — Context Management & Memory

- [ ] **CONTEXT-01**: System shall implement relevance filtering to reduce token usage by 50%+ vs. unbounded context
- [ ] **CONTEXT-02**: System shall maintain separate working memory (current conversation) and long-term memory (historical data)
- [ ] **CONTEXT-03**: System shall implement automatic context compaction when token budget exceeds threshold
- [ ] **CONTEXT-04**: System shall preserve critical information during compaction (goals, preferences, key events)
- [ ] **CONTEXT-05**: System shall support configurable token budgets per use case (combat, exploration, conversation)
- [ ] **CONTEXT-06**: System shall implement rolling summary updates without losing important details

### GOAL — Goal-Directed Behavior

- [ ] **GOAL-01**: System shall support natural language goal setting ("explore the dungeon", "level up to 10")
- [ ] **GOAL-02**: System shall decompose high-level goals into actionable subgoals automatically
- [ ] **GOAL-03**: System shall track progress toward goals with measurable milestones
- [ ] **GOAL-04**: System shall detect and report goal completion or failure
- [ ] **GOAL-05**: System shall maintain goal state across sessions (persistent goals)
- [ ] **GOAL-06**: System shall prioritize actions based on active goals

### PREF — Preference Learning

- [ ] **PREF-01**: System shall capture user feedback on agent decisions (explicit corrections)
- [ ] **PREF-02**: System shall learn preferences implicitly from user overrides
- [ ] **PREF-03**: System shall assign confidence scores to learned preferences
- [ ] **PREF-04**: System shall summarize preferences into actionable rules for LLM
- [ ] **PREF-05**: System shall persist preferences across sessions
- [ ] **PREF-06**: System shall apply learned preferences to future similar decisions

### DIALOG — Multi-Turn Conversations

- [ ] **DIALOG-01**: System shall maintain conversation state for NPC interactions (minimum 10 turns)
- [ ] **DIALOG-02**: System shall detect dialogue acts (greeting, question, command, farewell)
- [ ] **DIALOG-03**: System shall track conversation topics and transitions
- [ ] **DIALOG-04**: System shall preserve conversation context across interruptions
- [ ] **DIALOG-05**: System shall support multiple concurrent NPC conversations
- [ ] **DIALOG-06**: System shall summarize completed conversations for long-term memory

## Out of Scope

| Requirement | Reason |
|-------------|--------|
| **Full RLHF training** | Requires labeled dataset, over-engineering for single-user client |
| **Cloud-based memory** | Single-player focus, local storage sufficient |
| **Multi-agent coordination** | Out of scope for single-player MUD client |
| **Real-time collaboration** | Contradicts autonomous agent design |
| **Video/audio input** | Text-only MUD interface |
