  <action>
Create preference_manager.py with:

1. **PreferenceCategory enum** with values: LOOT, EQUIPMENT, MOVEMENT, CONVERSATION, GENERAL

2. **Preference dataclass** with fields:
   - `id: str` — unique identifier (category_rule hash)
   - `category: PreferenceCategory` — one of the 5 categories
   - `rule: str` — natural language description of the preference
   - `confidence: float` — Bayesian confidence score 0.0-1.0
   - `evidence_count: int` — number of supporting instances
   - `created_at: float` — timestamp from time.time()
   - `last_seen: float` — timestamp for pruning logic

3. **Methods for serialization**:
   - `to_dict()` — returns dict for JSON serialization (category as string)
   - `from_dict(d)` — classmethod to reconstruct from dict

4. **Bayesian confidence update methods**:
   - `agree()` — increase confidence: new_confidence = confidence + (1 - confidence) * 0.2, bounded to 1.0
   - `disagree()` — decrease confidence: new_confidence = confidence * 0.8, bounded to 0.0
   - `touch()` — update last_seen timestamp to current time
   - `is_stale(days: int = 30, threshold: float = 0.1) -> bool` — returns True if last_seen > days ago AND confidence < threshold

Per D-03: Bayesian updates — increase on agreement, decrease on contradiction, bounded 0-1.
Per D-04: Pruning — confidence < 0.1 after 30 days.

from preference_manager import Preference, PreferenceCategory
import time

# Test basic creation

p.disagree()  # Decrease

p.agree()

p.confidence = 0.01
p.disagree()
assert p.confidence == 0.0  # Capped at 0.0

Extend preference_manager.py with PreferenceManager class:

1. **Constructor**: `__init__(self, preferences_file: str = "preferences.json")`
   - Initialize empty preferences dict: `{id: Preference}`
   - Store preferences_file path
   - Call `load_preferences()`

2. **Core CRUD methods**:
   - `create_preference(category: PreferenceCategory, rule: str, confidence: float = 0.5) -> Preference` — create new preference, add to dict, save
   - `get_preference(pref_id: str) -> Optional[Preference]` — find by ID
   - `update_preference(pref_id: str, **kwargs) -> Optional[Preference]` — update rule or confidence
   - `delete_preference(pref_id: str) -> bool` — remove from dict, save
   - `list_preferences(category: Optional[PreferenceCategory] = None) -> List[Preference]` — return all, optionally filtered

3. **Evidence management**:
   - `add_evidence(pref_id: str, positive: bool = True) -> Optional[Preference]` — call agree() or disagree(), increment evidence_count, touch(), save
   - `get_preference_for_action(category: PreferenceCategory, action: str) -> Optional[Preference]` — fuzzy match rule against action text

4. **Persistence methods**:
   - `save_preferences()` — write all preferences to preferences.json, prune stale
   - `load_preferences()` — load from preferences.json at startup, create empty file if not exists
   - `prune_stale(max_age_days: int = 30, threshold: float = 0.1)` — remove preferences where is_stale() returns True

5. **State callbacks**:
   - `set_on_change_callback(callback)` — callback invoked on any preference state change

6. **ID generation**:
   - Generate stable ID from category + rule hash: `f"{category.value}_{hash(rule) % 100000}"`
   - Check for existing preference with same category + similar rule before creating

7. **Display formatting**:
   - `format_summary() -> str` — returns natural language summary: "Agent knows you prefer: [rule] (confidence: 87%, 5 examples)" per preference

Per D-03: Bayesian confidence updates, weighted 0.5 for implicit vs 1.0 for explicit.
Per D-04: Pruning stale preferences.
Per D-05: Display format in natural language.
