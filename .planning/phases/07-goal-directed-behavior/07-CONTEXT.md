# Phase 7: Goal-Directed Behavior - Context

**Gathered:** 2026-04-14
**Status:** Ready for planning

<domain>
## Phase Boundary

This phase delivers goal-directed agent behavior: users set natural language goals via WebSocket, the LLM agent decomposes them into actionable subgoals, tracks progress after each MUD output cycle, and persists goals across sessions. Users can view goal progress and the system automatically detects completion or failure.

</domain>

<decisions>
## Implementation Decisions

### Goal Input & Storage
- Users set goals via natural language string sent as WebSocket command — consistent with existing protocol
- Multiple active goals supported in ordered priority list — matches "prioritizes based on active goals"
- Goals stored on `LLMAgent.active_goals` list (already seeded in Phase 6 ContextManager) — reuse existing
- Goal data: name, description, status (active/complete/failed), created_at — minimal, extensible

### Subgoal Decomposition
- Subgoals generated by LLM call with goal + current game state → structured list of actionable subgoals
- 3-5 subgoals max per goal — focused, manageable
- Subgoals represented as simple list of strings with completion status — minimal overhead
- Subgoals refreshed on progress event + when current subgoal completes — event-driven

### Progress Tracking & Display
- Goal completion detected by LLM evaluating game state against goal description — intelligent, flexible
- Progress updates triggered after each MUD output cycle — natural cadence matching existing play loop
- Progress shown via WebSocket broadcast on goal update — consistent with existing broadcast pattern
- Goal update message format: `{"type": "goal_update", "goals": [...], "active_subgoal": "..."}` — follows existing JSON protocol

### Goal Persistence
- Goals saved on every goal state change — consistent with Phase 5 JSON persistence pattern
- Goals saved to `goals.json` in project root — separate, clear naming
- Keep last 20 completed goals — useful for LLM context reference
- Goals loaded at startup from goals.json — cross-session persistence

</decisions>
